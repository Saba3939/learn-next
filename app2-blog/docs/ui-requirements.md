# アプリ2: シンプルブログ - UI要件詳細

## 概要

このドキュメントでは、ブログアプリの各画面・コンポーネントのUI要件を詳細に定義します。
`requirements.md`のUI/UX要件を補完するもので、実装時の具体的な指針となります。

---

## 🎨 デザイントークン

### カラー

shadcn/ui の `neutral` カラースキームをベースとする。

| 用途 | クラス例 | 説明 |
|------|----------|------|
| テキスト（メイン） | `text-foreground` | 記事本文など |
| テキスト（サブ） | `text-muted-foreground` | 日付、補足情報 |
| 背景 | `bg-background` | ページ背景 |
| カード背景 | `bg-card` | カード・サイドバー |
| ボーダー | `border` | 区切り線 |
| 公開バッジ | `bg-green-100 text-green-800` | 公開済みステータス |
| 下書きバッジ | `bg-gray-100 text-gray-600` | 下書きステータス |

### タイポグラフィ

| 要素 | クラス例 | 用途 |
|------|----------|------|
| ページタイトル | `text-3xl font-bold tracking-tight` | h1 |
| セクション見出し | `text-xl font-semibold` | h2 |
| 記事タイトル（カード） | `text-lg font-semibold leading-snug` | カード内タイトル |
| 本文 | `text-base leading-relaxed` | 記事本文 |
| 補足テキスト | `text-sm text-muted-foreground` | 日付・カテゴリー |

### スペーシング

- セクション間: `py-8` または `py-12`
- カード内パディング: `p-6`
- フォームフィールド間: `space-y-4` または `space-y-6`
- ページ左右マージン: `container mx-auto px-4`

### ブレークポイント（Tailwind CSS デフォルト）

| 名前 | 幅 | 用途 |
|------|----|------|
| デフォルト（mobile） | < 640px | スマートフォン |
| `sm` | 640px〜 | 大きめスマートフォン |
| `md` | 768px〜 | タブレット |
| `lg` | 1024px〜 | デスクトップ |
| `xl` | 1280px〜 | ワイドデスクトップ |

## 🗂️ 共通レイアウト

### 公開ページ 共通ヘッダー

```
┌─────────────────────────────────────────────────────────┐
│  [ブログタイトル]                    [ナビゲーションリンク] │
│  (font-bold text-xl)                 ホーム  カテゴリー   │
└─────────────────────────────────────────────────────────┘
```

**仕様:**
- `sticky top-0 z-50` で常に上部固定
- 背景: `bg-background/80 backdrop-blur-sm border-b`
- コンテナ幅: `container mx-auto px-4`
- 高さ: `h-16`
- ブログタイトルは `/` へのリンク
- モバイルではナビゲーションを省略してもよい（ハンバーガーメニューはオプション）

### 公開ページ 共通フッター

```
┌─────────────────────────────────────────────────────────┐
│            © 2025 ブログ名. All rights reserved.         │
└─────────────────────────────────────────────────────────┘
```

**仕様:**
- `border-t mt-16 py-8`
- テキスト: `text-sm text-muted-foreground text-center`

```
┌─────────────────────────────────────────────────────────┐
│  🔧 管理画面                          [サイトに戻る]      │
```

**仕様:**
- ヘッダー: `bg-slate-900 text-white`、高さ `h-14`
- ナビゲーション: `border-b bg-muted/30`
- アクティブなナビリンク: `font-medium border-b-2 border-primary`
- コンテンツエリア: `container mx-auto px-4 py-8`

## 📄 公開ページ

### P1. 記事一覧ページ（トップページ `/`）

#### レイアウト

```
[Header]
┌──────────────────────────────────────────────────────────┐
│  全記事  最新のブログ記事                                  │
│  ─────────────────────────────────────────────────────── │
│  ┌─────────────────────────┐  ┌──────────────────────┐  │
│  │  記事一覧 (2/3カラム)    │  │ サイドバー (1/3)     │  │
│  │                         │  │                      │  │
│  │  [PostCard]             │  │  カテゴリー一覧       │  │
│  │  [PostCard]             │  │  ─────────────       │  │
│  │  [PostCard]             │  │  ・技術 (12)         │  │
│  │  ...                    │  │  ・デザイン (5)      │  │
│  │                         │  │  ・全て表示          │  │
│  │  [Pagination]           │  │                      │  │
│  └─────────────────────────┘  └──────────────────────┘  │
└──────────────────────────────────────────────────────────┘
[Footer]
```

**レイアウト仕様:**
- `grid grid-cols-1 lg:grid-cols-3 gap-8`
- 記事一覧: `lg:col-span-2`
- サイドバー: `lg:col-span-1`
- モバイルではサイドバーは記事一覧の下に表示

**ページタイトルエリア:**
- `mb-8` の余白
- タイトル: `text-3xl font-bold tracking-tight`
- サブテキスト（記事数・フィルター中の表示など）: `text-muted-foreground mt-2`

**カテゴリーフィルター:**
- アクティブカテゴリー時: ページタイトル下に `Badge` でフィルター中表示
- 「フィルターをクリア」リンクを表示

#### PostCard コンポーネント

```
┌──────────────────────────────────────────┐
│  [カテゴリーバッジ]                        │
│                                          │
│  記事タイトルがここに表示される              │
│  長い場合は2行で折り返し                   │
│                                          │
│  記事の抜粋テキストがここに表示される。      │
│  最大3行程度で切り捨て。                   │
│                                          │
│  2025年1月15日          続きを読む →      │
└──────────────────────────────────────────┘
```

**仕様:**
- shadcn/ui の `Card` (`CardHeader`, `CardContent`, `CardFooter`) を使用
- カード全体がリンクになる（`hover:shadow-md transition-shadow`）
- タイトル: `text-lg font-semibold leading-snug line-clamp-2`
- 抜粋: `text-sm text-muted-foreground line-clamp-3`
- フッター: 日付（左）と「続きを読む」（右）を `flex justify-between items-center`
- 「続きを読む」: `text-sm text-primary hover:underline`
- カテゴリーバッジは `CardHeader` 内の上部に配置

#### サイドバー: カテゴリー一覧

```
┌──────────────────────────┐
│  カテゴリー               │
│  ──────────────────────  │
│  すべて               (25)│
│  技術                 (12)│
│  デザイン              (5)│
│  ビジネス              (8)│
└──────────────────────────┘
```

**仕様:**
- `Card` コンポーネントを使用
- タイトル: `CardHeader` 内に `text-sm font-semibold uppercase tracking-wider text-muted-foreground`
- リスト: `space-y-1`
- 各アイテム: `flex justify-between items-center px-2 py-1.5 rounded-md text-sm hover:bg-muted cursor-pointer`
- アクティブなカテゴリー: `bg-muted font-medium`
- 記事数: `text-xs text-muted-foreground`

#### 記事グリッド

- デスクトップ: カードは縦に並べる（1列）または2列
- モバイル: 1列
- カード間の間隔: `space-y-6` または `grid gap-6`

#### ページネーション

```
                  ← 前へ  1  2  [3]  4  5  次へ →
```

**仕様:**
- `flex justify-center items-center gap-2 mt-8`
- 現在のページ: `Button variant="default"` (塗りつぶし)
- 他のページ: `Button variant="outline"`
- 前へ/次へ: `Button variant="ghost"` + ChevronLeft/Right アイコン
- ページ数が多い場合は `...` で省略（5ページ以上）
- 前のページなし/次のページなしのとき該当ボタンを `disabled`

**空の状態（記事がない場合）:**
```
┌──────────────────────────────────────┐
│                                      │
│      📄                              │
│  まだ記事がありません                 │
│  最初の記事を書いてみましょう！        │
│                                      │
│      [管理画面へ]                     │
│                                      │
└──────────────────────────────────────┘
```

#### レイアウト

```
[Header]
┌──────────────────────────────────────────────────────────┐
│  ← 記事一覧に戻る                                         │
│                                                          │
│  [カテゴリーバッジ]                                        │
│  # 記事タイトル（大きく）                                  │
│  📅 2025年1月15日                                         │
│  ────────────────────────────────────────────            │
│                                                          │
│  記事本文がここに表示される。段落ごとに                      │
│  適切な余白をとって読みやすく。                             │
│                                                          │
│  本文本文本文...                                          │
│                                                          │
│  ────────────────────────────────────────────            │
│  関連記事                                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ [Card]   │  │ [Card]   │  │ [Card]   │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└──────────────────────────────────────────────────────────┘
[Footer]
```

**ヘッダーエリア仕様:**
- 「← 記事一覧に戻る」: `text-sm text-muted-foreground hover:text-foreground flex items-center gap-1 mb-8`
- カテゴリーバッジ: タイトル上部に配置
- タイトル: `text-3xl md:text-4xl font-bold tracking-tight leading-tight mt-3`
- 日付: `text-sm text-muted-foreground flex items-center gap-1.5 mt-3`（アイコン付き）
- 区切り線: `Separator` コンポーネント、`my-8`

**本文エリア仕様:**
- `max-w-3xl`（読みやすい幅に制限）でセンタリング
- `prose` クラス（Tailwind Typographyを使う場合）または手動で `leading-relaxed text-base space-y-4`
- 段落間: `mb-4`

**関連記事エリア仕様:**
- `mt-16 border-t pt-8`
- タイトル: `text-xl font-semibold mb-6`
- グリッド: `grid grid-cols-1 md:grid-cols-3 gap-4`
- PostCard コンポーネントを再利用（compact variant 推奨）

**存在しない記事（404）:**
- Next.js の `notFound()` を使用 → `not-found.tsx` にカスタム404ページを実装
- 「記事が見つかりません」メッセージ + 「一覧に戻る」ボタン

#### レイアウト

```
┌──────────────────────────────────────────────────────────┐
│  記事管理                            [+ 新規記事作成]      │

│  ├────────┼──────────┼────────┼───────────┼──────────┤  │
│  │ 記事A  │ 技術      │[公開] │ 2025/1/10 │[編集][削]│  │
│  │ 記事B  │ デザイン  │[下書] │ 2025/1/8  │[編集][削]│  │
│  │ ...    │ ...       │ ...   │ ...       │ ...      │  │
│  └────────┴──────────┴────────┴───────────┴──────────┘  │
│                                                          │
│              ← 前へ  1  2  [3]  次へ →                   │
└──────────────────────────────────────────────────────────┘
```

**ページヘッダー仕様:**
- `flex justify-between items-center mb-6`
- タイトル: `text-2xl font-bold`
- 「+ 新規記事作成」ボタン: `Button` (default variant) + PlusIcon

**テーブル仕様 (shadcn/ui `Table`):**

| カラム | 幅の目安 | 内容 |
|--------|----------|------|
| タイトル | flex-1 | テキスト、長い場合 truncate。クリックで編集ページへのリンク |
| カテゴリー | 120px | カテゴリー名テキスト |
| 状態 | 80px | `Badge`（公開: green、下書き: gray） |
| 作成日 | 120px | `YYYY/MM/DD` フォーマット |
| 操作 | 120px | 編集・削除ボタン |

**操作ボタン仕様:**
- 編集: `Button size="sm" variant="outline"` + PencilIcon
- 削除: `Button size="sm" variant="outline"` の赤系 (`text-destructive hover:bg-destructive/10`)
- ボタン間: `gap-2`

**削除確認ダイアログ:**
```
┌─────────────────────────────────┐
│  記事を削除しますか？             │
│                                 │
│  「記事タイトル」を削除します。   │
│  この操作は元に戻せません。       │
│                                 │
│       [キャンセル]  [削除する]   │
└─────────────────────────────────┘
```
- `Dialog` コンポーネント使用
- 削除ボタン: `Button variant="destructive"`
- 削除中: ボタンを `disabled` + ローディングスピナー表示

**空の状態:**
```
記事がまだありません。
最初の記事を作成してみましょう！
[+ 新規記事作成]
```

### A2. 記事作成ページ（`/admin/posts/create`）

#### レイアウト

```
┌──────────────────────────────────────────────────────────┐
│  ← 記事管理に戻る                                         │
│  新規記事作成                                             │
│  ────────────────────────────────────────────────────── │
│                                                          │
│  タイトル *                                              │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 記事タイトルを入力...                               │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  スラッグ *                                              │
│  ┌────────────────────────────────────────────────────┐ │
│  │ my-post-title                                      │ │
│  └────────────────────────────────────────────────────┘ │
│  ℹ️ URLに使用されます。英数字とハイフンのみ使用可能です。    │
│                                                          │
│  カテゴリー *                                             │
│  ┌────────────────────────────────────────────────────┐ │
│  │ カテゴリーを選択...                          ▼      │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  抜粋                                                    │
│  ┌────────────────────────────────────────────────────┐ │
│  │                                                    │ │
│  │ (3行程度)                                          │ │
│  └────────────────────────────────────────────────────┘ │
│  0 / 200文字                                             │
│                                                          │
│  本文 *                                                  │
│  ┌────────────────────────────────────────────────────┐ │
│  │                                                    │ │
│  │                                                    │ │
│  │ (8行以上の大きなテキストエリア)                      │ │
│  │                                                    │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  公開設定                                                │
│  下書き  ●────  公開                                     │
│  公開すると記事が一般に公開されます。                      │
│                                                          │
│                       [キャンセル]  [記事を保存する]      │
└──────────────────────────────────────────────────────────┘
```

**フォーム仕様:**
- 最大幅: `max-w-3xl mx-auto`
- フィールド間: `space-y-6`
- 必須フィールドには `*` マーク（ラベル右に `text-destructive`）

**各フィールド仕様:**

| フィールド | コンポーネント | バリデーション表示 |
|-----------|--------------|-----------------|
| タイトル | `Input` | エラーは入力欄の下に `text-sm text-destructive` |
| スラッグ | `Input` + ヒントテキスト | タイトル入力時に自動生成（日本語はローマ字変換なしでそのまま除去） |
| カテゴリー | `Select` | エラーは選択肢の下に表示 |
| 抜粋 | `Textarea` rows=3 | 文字数カウンター表示 (`text-xs text-muted-foreground text-right`) |
| 本文 | `Textarea` rows=10 | 最小高 `min-h-[200px]` |
| 公開設定 | `Switch` + `Label` | スイッチ右にステータステキスト |

**スラッグの自動生成:**
- タイトル入力時にリアルタイムで slug を生成（`onChange`）
- 一度スラッグを手動編集したら自動生成を停止
- ヒントテキスト: `text-xs text-muted-foreground mt-1`

**送信ボタンエリア:**
- `flex justify-end gap-3 mt-8 pt-6 border-t`
- キャンセル: `Button variant="outline"`（記事管理一覧に戻る）
- 保存: `Button` (default)
- 保存中: ボタン内にスピナー + `disabled`

**成功時:**
- 記事管理ページへリダイレクト
- Toast通知: 「記事を作成しました」（成功）

**エラー時:**
- バリデーションエラー: 各フィールド下にインライン表示
- サーバーエラー: フォーム上部に `Alert` コンポーネントで表示

**テーブル仕様:**

| カラム | 内容 |
|--------|------|
| カテゴリー名 | テキスト |
| スラッグ | `font-mono text-sm text-muted-foreground` |
| 記事数 | 「N件」形式。0件の場合は記事がある場合のみ削除可能 |
| 操作 | 編集・削除ボタン |

**編集ダイアログ仕様:**
```
┌──────────────────────────────────────┐
│  カテゴリーを編集                     │  ✕
│                                      │
│  カテゴリー名 *                       │
│  ┌────────────────────────────────┐  │
│  │ 技術                           │  │
│  └────────────────────────────────┘  │
│                                      │
│  スラッグ *                           │
│  ┌────────────────────────────────┐  │
│  │ tech                           │  │
│  └────────────────────────────────┘  │
│                                      │
│  説明                                │
│  ┌────────────────────────────────┐  │
│  │                                │  │
│  └────────────────────────────────┘  │
│                                      │
│             [キャンセル]  [保存する]  │
└──────────────────────────────────────┘
```

**削除確認ダイアログ仕様:**
- 記事が紐づいている場合:
```
┌──────────────────────────────────────┐
│  カテゴリーを削除できません           │
│                                      │
│  「技術」カテゴリーには12件の記事が  │
│  紐づいているため削除できません。    │
│  先に記事を別のカテゴリーに移動      │
│  するか、削除してください。          │
│                                      │
│                       [閉じる]       │
└──────────────────────────────────────┘
```

- 記事が紐づいていない場合:
```
┌──────────────────────────────────────┐
│  カテゴリーを削除しますか？           │
│                                      │
│  「デザイン」を削除します。          │
│  この操作は元に戻せません。          │
│                                      │
│       [キャンセル]  [削除する]       │
└──────────────────────────────────────┘
```

#### ボタンレベル
- フォーム送信中: `<Loader2 className="animate-spin" />` アイコン + テキスト変更
  - 例: 「保存する」→「保存中...」
- ボタンを `disabled` にして二重送信防止

- 「再試行」ボタンを提供

```
│                                      │
```

- 失敗: `variant="destructive"`
- 表示時間: 3〜5秒

## ♿ アクセシビリティ

- フォームの `<label>` と `<input>` は正しく紐付ける（`htmlFor` / `id`）
- エラーメッセージは `aria-describedby` で入力欄と紐付ける
- ダイアログは `role="dialog"` + `aria-modal="true"` + `aria-labelledby`（shadcn/ui が対応済み）
- アイコンのみのボタンには `aria-label` を設定
  - 例: 削除ボタン `<Button aria-label="記事を削除">`
- キーボードナビゲーション: Tab キーでフォーカス移動できること
- カラーコントラスト比: WCAG AA 基準（4.5:1 以上）を満たすこと
